## The Technical

*The specialist. The one that speaks in code, coordinates, and data.*

### The Job

The Technical voice handles code blocks, terminal output, data readouts, syntax-highlighted content, and any context where precision and alignment matter more than reading flow. It is the system's most constrained voice — monospace by nature, optimized for environments where every character must occupy the same width and where color (syntax highlighting, status indicators, data categories) is a primary layer of meaning on top of the letterforms.

### The Character

Monospace. Highly legible — the most readable voice in the system when clarity of individual characters is the measure. Clear differentiation between easily confused glyphs: 0/O, 1/l/I, rn/m. This is not aesthetic preference — it is functional necessity. Ambiguity in the Technical voice is a defect, not a style choice.

Clean and modern in construction, but not so stripped down that it loses personality. The Technical voice carries the Technical end of the Technical↔Human tension — but even at the technical extreme, the system's instinct is that the machine has a heartbeat. A monospace face with considered curves, optical corrections, and craftsmanship in the details — not a raw bitmap grid, and not a bland system monospace.

### Relationship with Color

The Technical voice is designed to coexist with color as a functional layer. Syntax highlighting, status colors, categorical color coding — the letterforms must remain legible and harmonious across a range of foreground colors against both light and dark backgrounds. This means the stroke weight, character width, and internal proportions must be robust enough that a green keyword, an orange string, and a gray comment all remain equally readable.

**This is a selection criterion:** When evaluating a monospace typeface for the Technical voice, it should be tested with a representative syntax theme, not just in monochrome.

### Scale and Behavior

The Technical voice operates in a narrower range than the other two. It is most at home at small to medium sizes — code editors, terminal windows, data tables. It can be pushed to display sizes for stylistic purposes (a hero element showing a code snippet, a decorative terminal readout), but this is an occasional treatment, not its primary mode. At small sizes, it must remain crisp and unambiguous.
